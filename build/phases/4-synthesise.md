# Phase 4 — Synthesise (Planner)

Incorporate all review feedback into the final PLAN.md. This is the version that gets built.

## Startup

1. Read `PLAN.md` — the Phase 1 draft
2. Read `progress/build-review.md` — Researcher and Reviewer findings (may not exist in fast mode)

## Process

1. Apply all critical and major findings from the reviews to PLAN.md
2. Revise the approach section if the reviews suggested a better one
3. Refine task breakdown if reviews identified missing tasks or incorrect scope
4. Update acceptance criteria where reviews found them insufficient
5. If reviews were clean or fast mode was used — confirm PLAN.md as-is

## Final Check

Before proceeding to Phase 5, confirm:
- [ ] Every task has testable acceptance criteria
- [ ] Tasks are ordered by dependency
- [ ] No task is doing two unrelated things
- [ ] Open Questions are documented

## Handoff

Print:
```
PLAN.md finalised. [N] tasks ready for implementation.

Proceeding to Phase 5 — Tester will write failing tests before any code is written.
```

Do not start Phase 5 until this confirmation is printed.
