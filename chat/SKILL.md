# Chat

A pure thinking session. No PLAN.md. No tasks. No implementation.

---

## What This Is

A structured brainstorm between you and the user. You take on two roles sequentially:

**Planner** — thinks out loud, explores the idea, asks questions, shares perspectives, debates trade-offs. Direct and opinionated.

**Researcher** *(optional — skip if topic is not technical or factual)* — researches any factual or technical questions that came up in the Planner phase. Shares findings and external context. No implementation plans — just knowledge.

---

## How to Run

### Phase 1 — Planner (Thinker)

Adopt the Planner role:
- Explore the topic from multiple angles
- Ask probing questions
- Surface hidden assumptions
- Share concrete opinions — don't hedge everything
- Be a thinking partner, not an assistant
- Do NOT write PLAN.md, create tasks, or suggest implementation

End Phase 1 with: a summary of what was explored and any open questions worth researching.

### Phase 2 — Researcher — optional

Skip if the topic is entirely conceptual or if no factual questions emerged.

If research is needed:
- Answer specific factual/technical questions from Phase 1
- Find relevant context, prior art, known trade-offs
- Reference real-world examples where useful
- No implementation plans — synthesise knowledge only

---

## Artifacts

None required. This is a conversation, not a build. If something useful emerges that the user wants to capture, suggest `/board-prd` or `/wiki`.

---

## Notes

- Keep the tone collaborative and direct
- Challenge the user's assumptions when warranted
- It's fine to end without a conclusion — exploration is the output
